# 메타데이터 추출 규칙

## 1. 요약(summary) 작성 규칙

- 2-3문장으로 문서의 핵심 내용을 요약
- 문서의 목적, 주요 주제, 핵심 결론 포함
- 전문 용어는 유지하되 맥락 설명 포함
- 한국어 문서는 한국어로, 영어 문서는 한국어로 요약

## 2. 카테고리(categories) 분류 규칙

다음 카테고리 중 해당하는 것을 모두 선택 (1-3개):

| 카테고리 | 설명 |
|---------|------|
| `기술문서` | API 문서, 기술 스펙, 아키텍처 설계 |
| `가이드` | 사용법, 튜토리얼, How-to |
| `정책/규정` | 사내 정책, 규정, 프로세스 |
| `보고서` | 분석 보고서, 리서치, 성과 보고 |
| `회의록` | 미팅 노트, 의사결정 기록 |
| `제안서` | 프로젝트 제안, 기획안 |
| `데이터` | 데이터셋 설명, 스키마 정의 |
| `기타` | 위 분류에 해당하지 않는 경우 |

## 3. 태그(tags) 추출 규칙

- 3-7개의 관련 키워드 추출
- 문서에서 반복되는 핵심 개념 우선
- 고유명사(제품명, 서비스명, 기술명) 포함
- 일반적인 단어(문서, 파일 등)는 제외
- 한글 태그 우선, 필요시 영문 병기

## 4. 파일 타입별 처리

### PDF
- 텍스트 레이어 우선 분석
- 표, 차트가 있으면 데이터 요약 포함

### DOCX
- 제목/소제목 구조 활용
- 목차가 있으면 구조 파악에 활용

### TXT/MD
- 마크다운 헤딩 구조 활용
- 코드 블록은 기술 태그로 추출

### CSV
- 컬럼명으로 데이터 성격 파악
- 데이터 샘플로 내용 유추
